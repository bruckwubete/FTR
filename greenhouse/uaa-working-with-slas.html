      Working with SLAs

       

      UAA Configuration >SLAs helps operator to identify services that are conforming or violating the service level agreements and also services that are in warning state.

       

      Follow the given steps to create a new SLA:

      1. Open Configuration >SLAs from the Navigation Menu.

      2. Click + to create a new SLA icon from the toolbar.

      3. Enter the details of each tab as per the table below. Click Add and Close .

      Conformance Objectives Tab

       

      SLAs - Creating a new SLA - Conformance Tab

      Name

      Enter a Name for the conformance objective.

      Conformance Type

      Select the Conformance Type from the drop-down list -

      • Conformance %
      • Avg
      • Sum
      • Min
      • Max

      Conformance State

      Select the High and Low values for Violated - Conformance State

      Conformance Condition
      • Add appropriate Conformed state and Warning state values for Target % value Property
      • Select (Warning, Minor, Major, Critical) from the Conformed state drop-down and Warning state drop-down for Alarm Severity Property
      • For Conformance Type except Conformance % , you need to add at least one of the Value High or Value Low for Conformed state and Warning state

      • Make sure that the values of Conformance / Warning - Value High and Value low follows the given order -

        • Conformance State high >Waring state high >Warning State low >Conformance State low

      Formula

      Define the formula used to compute the conformance value using the metric variables defined in the Metric Variables table.

      You can change the order of the column how it has to appear in the Conformance Objectives table, by using the First, Up, Down and Last icons available on the right-hand side of the pane.

       

      To edit a SLA, you can simply select the desired SLA from the Configuration >SLAs list and click the edit icon on the toolbar.
