      Working with service templates

       

      UAA Configuration >Service Templates lists all the Monitoring Templates created in the system.

       

      Follow the given steps to create new Service Template:

      1. Open Configuration >Service Templates from the Navigation Menu.

      2. Click + icon to create a new Service Template.

      3. Enter the details of each tab as per the table below. Click Add and Close .
       

      Column Name

      Description

      Name

      The name of the template.

      Description

      Description of the Service.

      Domain

      Select the Domain to which the service template belongs.

      Service type

      Select a Service type. When a Service type is selected, the metrics associated with the service type will be assigned to the template.

      Datasource Expression

      The Datasource expression is used in service templates to auto-select the data sources to service from assigned endpoints, mainly through import or service discovery.

      To Create a Service Template with Datasource Expression:

      1. Create one Service Template with DataSource Expression and use the same Service Template in Services while creating Services. (OR you can also select the Service Template which contains DataSource Expressions while creating Services).
      2. Now, assign a few Endpoints from Endpoints Tab in Services ADD page for the same Services while creating.
      3. Navigate to Datasource Tab, to enable Collect Performance checkbox in the same Services page.
      4. Now you can view, the Datasource Tab is auto populated the Datasources in Included Datasource Table based on the Datasource Expression in the added service template.

      SLA Conformance

      Define SLA Conformance from the Conformance tab

      Availability

      A service is available if it is up and working regardless of the quality or conformance of the service. Here, service availability could be determined from alarms. Availability state will be Unknown at the time of creation. Once processed, the status will be changed to Available if service is not affected.

      When there is no Availability defined for a service, the Availability state will be shown as Unknown for the service. Different Availability states can be -

      • Unknown
      • Available
      • Unavailable
      • At risk

      Availability of a service is calculated based on -

      • Event Policy
      • Aggregation Policy

      Event Policy

      A service can be Available, Unavailable or At risk based on the event condition specified for a service.

      Enable Aggregation Policy

      Aggregation policy calculation takes place every one minute. Aggregation policy is based on the availability of child services. We have two options -

      • Any child
      • Percentage of child

      If you select Any child, then you can specify as the parent service will be At risk, if any of the child is at risk or unavailable.

      Service Unavailable, if any child is Unavailable.

      If you select the percentage aggregation then mention the percentage of child to be unavailable or At risk to make a service unavailable or at risk. Decimals are not allowed.

      If mentioned percentage of child services are Unavailable or At risk, the service will become Unavailable or At risk, as defined in the aggregation policy.

      When an Event policy and an aggregation policy both are defined for a service, then the most critical condition takes the priority and gets applied to the service.

      For example, a service has both of the following conditions meeting -

      • Event policy: Unavailable if there is a critical alarm
      • Aggregation policy: At risk if any child is at risk.

      Then the service will be Unavailable.

      Carry Node Availability

      A boolean field (check box) is added which states whether to consider the node’s availability in service’s availability status. If enabled, node’s availability will be considered in service availability calculation and the node availability state will be propagated to the service.

      Quality

      Service quality is a measure of how well the service is performing. It is derived entirely from performance data. All measures of service quality (or Key Quality Indicators) are defined as metrics on the service. There could be any number based on what the service is.

      The overall quality is then a summary of the metrics which is simply defined as a grade.

      Performance Metrics Tab

       

      The Performance Metrics tab is used to add metric references to a service template. Performance metrics tab displays all the service metrics and metric of metrics available in the system, that is: metrics with protocol 'service' and 'metric'.

      The performance computation frequency is derived from maximum pm collection frequency of selected data sources.

      The last poll value of the data sources should be used to calculate EVC PM.

      All Service Templates are assigned PMs related to the service's Health, Significance, and Urgency by default.

      Display order metrics tab

       

      The details for Display order metrics tab can be found here .

      Composite or Metric Group are not included in the display order metrics of the service template.

       

      To edit a service template, you can simply select the desired service template from the Configuration >Service Templates list and click the edit icon on the toolbar.

      Reverting a  service  template

      The Revert Template option allows you to revert the version of the template to the base version that is shipped with Software/RA.
